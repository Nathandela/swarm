# Centralized Agent Session Management: Landscape Report

**Scope:** Anthropic's Agent View, reverse-engineering work on its internals, cross-CLI alternatives, and mobile remote control.
**Date:** 16 July 2026.
**Purpose:** Reference for building a similar system.

---

## 1. The reference implementation: Agent View

The system shown in the source screenshot is **Agent View**, opened with `claude agents` (or the left arrow key inside a session). Anthropic shipped it on 11 May 2026 in Claude Code v2.1.139 as a research preview. Sessions are grouped into "Needs input", "Ready for review", and "Completed".

Primary source: [code.claude.com/docs/en/agent-view](https://code.claude.com/docs/en/agent-view).

### Documented architecture

| Component | Implementation |
|---|---|
| Process model | Per-user **supervisor daemon**, separate from the terminal, hosting detached sessions |
| State | `~/.claude/daemon/roster.json` (running sessions, for reconnect) and `~/.claude/jobs/<id>/state.json` (per-session) |
| Latency | Pre-warmed worker pool: one warm worker per dispatch, replacement spawned after |
| Lifecycle | Idle around 1 hour, process killed, transcript kept, restarted on next attach |
| Status rows | One-line summaries generated by a Haiku-class model |
| Isolation | Sessions relocate to `.claude/worktrees/<id>/` before writing files |

Dispatch: `claude --bg "task"` from the shell, or `/bg` inside a session.

Related third-party analyses: [Cobus Greyling's teardown](https://cobusgreyling.medium.com/claude-code-agent-view-703491634ea7) (frames it as harness, not model) and the [ClaudeFast guide](https://claudefa.st/blog/guide/agents/agent-view) (peek and attach flows).

## 2. Reverse-engineering the internals

Two distinct bodies of work exist.

### 2.1 Deobfuscation of the shipped bundle

Claude Code ships as a minified, not obfuscated, `cli.js`, so the logic has always been extractable. AfterPack extracted around 148,000 plaintext strings from the roughly 13 MB file, including system prompts and `tengu_`-prefixed telemetry events.

- [ghuntley.com/tradecraft](https://ghuntley.com/tradecraft/): full cleanroom transpilation of the minified JavaScript back into readable TypeScript. Best methodology reference.
- [Yuyz0112/claude-code-reverse](https://github.com/Yuyz0112/claude-code-reverse): visualization of Claude Code's LLM interactions. The author's thesis is that replicating the request and response patterns matters more than the code itself.
- [kirshatrov.com/posts/claude-code-internals](https://kirshatrov.com/posts/claude-code-internals): mitmproxy interception of API traffic.
- [BrightCoding deep dive](https://www.blog.brightcoding.dev/2025/07/17/inside-claude-code-a-deep-dive-reverse-engineering-report/): names internal constructs (`nO` master loop, `h2A` async message queue).
- [DEV.to version-tracking series](https://dev.to/kolkov/we-reverse-engineered-12-versions-of-claude-code-then-it-leaked-its-own-source-code-pij): describes the daemon-service pattern from the outside, including graceful degradation to standalone mode when the daemon is unavailable.

### 2.2 The 31 March 2026 source-map leak

Anthropic shipped v2.1.88 with a 59.8 MB source map exposing roughly 512,000 lines of TypeScript across around 1,900 files. The package was pulled within hours, but mirrors persist. This is the richest source on the daemon internals because it exposed the unreleased precursors to Agent View.

- [Satapathy's architecture breakdown](https://www.sidharthsatapathy.com/blog/claude-code-source-leak-architecture-analysis/): 12-phase bootstrap, with phases 6, 8, and 9 covering the daemon and background-session system.
- [Flowpad fact-check](https://flowpad.ai/blog/claude-code-source-leak): use this for calibration. It confirms the 44 feature flags and swarms, flags KAIROS as overstated, and identifies some claimed features as invented. Relevant flags: `DAEMON`, `BRIDGE_MODE`, `SSH_REMOTE`, `DIRECT_CONNECT`, `UDS_INBOX` (Unix domain socket IPC), and `COORDINATOR_MODE`.
- [The New Stack](https://thenewstack.io/claude-code-source-leak/): frames the whole as an "agent operating system".
- [Reimplementation walkthrough](https://dreadheadio.github.io/claude-code-roadmap/claude-code-roadmap-blog.html): rebuilds the cron-agent, swarm, and webhook features on public APIs. Key finding: inter-agent message passing is structured text through the coordinator's context window, with no dedicated bus.

**Caveat:** the leaked code is copyrighted internal material, and mirrors became a [malware vector](https://www.zscaler.com/blogs/security-research/anthropic-claude-code-leak). Study the analyses, not a random mirror.

## 3. Cross-CLI session managers

These are agent-agnostic: they spawn any CLI and track all of them centrally. Nearly all are **tmux-backed** rather than daemon-native. Tmux keeps sessions alive independently of the terminal, and the dashboard adds status detection by parsing each agent's output. This is more general than Anthropic's supervisor, since it wraps whatever CLI it is pointed at.

| Tool | Language | Agents | Distinguishing feature |
|---|---|---|---|
| [agent-of-empires](https://github.com/agent-of-empires/agent-of-empires) | Go | 13+ (Claude, Codex, Gemini, Cursor, Copilot, OpenCode, Pi, Qwen, and more) | TUI plus installable web PWA, auto-detects installed CLIs, Docker sandboxing |
| [agent-deck](https://github.com/asheshgoplani/agent-deck) | Go | Claude, Gemini, Codex, OpenCode, custom | "Conductor" supervisor session, 2-second polling, Telegram/Slack escalation |
| [thurbox](https://thurbox.thurbeen.eu) | - | Claude, Codex, opencode, Aider, Copilot, **any CLI you define** | Inter-session messaging, native code-review view, remote SSH |
| [claude-squad](https://github.com/smtg-ai/claude-squad) | Go | Claude, Codex, Gemini, Aider | The origin project most others descend from |
| [tmuxcc](https://github.com/nyanko3141592/tmuxcc) | Rust | Claude, OpenCode, Codex, Gemini | Clean per-agent parser modules. Best small reference to read |
| [ccmux](https://github.com/skzv/ccmux) | - | Claude, Codex, Cursor | Daemon holds a caffeinate lock, so work continues with the lid closed |
| [dux](https://www.patrickdap.com/post/how-to-run-multiple-agents/) | Rust | Any CLI matching the pattern | Wraps the official CLIs, not the APIs. Preserves each tool's own skills |
| [izll/agent-session-manager](https://github.com/izll/agent-session-manager) | Go | Claude | Cleanest separation of `instance.go` (PTY), `storage.go`, `status_detector.go` |

Curated indexes, updated frequently: [awesome-cli-coding-agents](https://github.com/bradAGI/awesome-cli-coding-agents) and [awesome-agent-orchestrators](https://github.com/andyrewlee/awesome-agent-orchestrators).

**Caveat:** "agent-agnostic" means the dashboard is generic while status detection is per-agent. Each CLI needs its own output parser or hook. [Marc Nuri's write-up](https://blog.marcnuri.com/ai-coding-agent-dashboard) describes the clean version of this pattern: a generic core, plus per-agent hooks and enrichers fed by heartbeats.

## 4. Mobile remote control

Purpose-built native apps now exist, so SSH into tmux on a small screen is no longer the only path.

| Tool | Platform | Agents | Notes |
|---|---|---|---|
| [AgentsRoom](https://agentsroom.dev/claude-code-remote) | iOS, Android (React Native/Expo) | Claude, Codex, Gemini, OpenCode, Aider | Widest roster. E2EE (Curve25519 plus AEAD), QR pairing, full xterm.js terminal, push notifications, live dev-server preview, worktrees |
| [Orca](https://github.com/stablyai/orca) | iOS, Android | Claude, Codex, Cursor | **Open source and self-hostable.** Auditable, no relay trust required |
| [Tactic Remote](https://tacticremote.com) | iPhone, iPad | Claude, Codex, Amp | Sessions organized by repo and branch. Built for running different agents on different worktrees of one problem |
| [CC Pocket](https://dev.to/k9i/i-built-a-mobile-app-to-control-claude-code-and-codex-from-my-phone-4d84) | Flutter | Claude, Codex | Approval-first design: approve/deny on the session list. Strong diff review (text plus three image modes). Bridge server plus Tailscale |
| [Grass](https://codeongrass.com/blog/best-app-to-control-coding-agents-from-mobile/) | - | Multiple | The multi-agent option offering **session persistence without the laptop on** |
| [Happy Coder](https://happy.engineering) | iOS, Android | Claude only | MIT, free, E2EE, self-hostable, voice coding |
| Claude Remote Control / Dispatch | Claude mobile app | Claude only | Zero setup, pairs directly with Agent View. See the [multi-device guide](https://radar.firstaimovers.com/claude-code-remote-control-dispatch-multi-device-guide-2026) |

Two decision axes:

1. **Where the agents run.** AgentsRoom, Tactic Remote, CC Pocket, and Happy are front-ends over a relay to a bridge on your machine. Agents stop if that machine sleeps or the desktop app closes, the same limitation Agent View has. Grass and cloud-VM setups avoid this.
2. **Trust.** E2EE relays (AgentsRoom, Happy) prevent the relay reading your code. Open source and self-hosted (Orca, Happy) removes the question entirely.

**Terms-of-service note:** some maintainers report Anthropic restricting accounts for certain third-party tooling, and Claude Pro/Max subscriptions are not permitted in some tools such as OpenCode (API key only there). Driving the official CLI from the outside, as the tmux wrappers do, is generally the safer side of that line. Verify per tool.

## 5. Build synthesis

Minimum viable architecture, combining Anthropic's documented design with what the clones actually do:

1. **Long-lived daemon** owning the session registry.
2. **Detached worker processes**, each in a PTY, running your own agent loop or a wrapped CLI.
3. **IPC channel** (Unix domain socket) between a thin TUI/CLI client and the daemon. This is Anthropic's `UDS_INBOX`.
4. **Persisted roster and per-session state** on disk, so the daemon can crash, restart, and reconnect.
5. **Git-worktree isolation** per session for parallel writes.
6. **Status detector.** Anthropic uses a Haiku call. The clones parse the PTY output stream, which is cheaper and works across agents.
7. **Detach and reattach** by keeping the transcript on disk and re-spawning a process against it.

**Key upstream difference:** the clones wrap the existing `claude` CLI, whereas Anthropic's supervisor spawns first-party sessions. If building an independent agent, start from `izll/agent-session-manager` (session, PTY, and state layer), `tmuxcc` (per-agent parsers), and the official agent-view docs (supervisor semantics).
