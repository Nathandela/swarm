# Audit 009 — Epic 11 (Claude Code + Codex adapters + status wiring)

**Date**: 2026-07-17
**Committee**: codex GPT-5.6 sol (cross-model) + Opus (independent, ran the FULL -race suite + gates). agy quota-blocked.
**Verdict**: FIX REQUIRED. Strong convergence: the adapters are conformant + the generic wiring is well-built, but the FEATURE (launch a real Claude/Codex session; its hooks drive status; resume works) is NOT delivered — the frozen tests pass only via the fake agent + hand-forged callbacks. -race green does NOT clear the two logical races.

## Consensus — FIX
1. **SetStatus process-guard TOCTOU resurrection race** (codex CRITICAL, Opus HIGH): daemon.go SetStatus reads meta under d.mu, releases, then saveMeta writes the STALE process dimension. Interleave: SetStatus reads running → handleShimExit/markLost persists exited/lost → SetStatus overwrites back to running. A dead session shows running until the next restart. EndSession doesn't take emitMu, so an in-flight emit sails past. Fix: atomic read-modify-write of the status against the CURRENT meta under one serialization boundary (re-read the live process dimension in the write critical section), or refuse to persist a non-running session.
2. **Typed status dies after a daemon restart** (both HIGH): RegisterSession is called ONLY from fresh launch; reconcile re-adopts sessions but never re-registers them with the engine, and the token is unpersisted → the rebuilt engine knows no token → every real hook is rejected as unregistered AND OnOutput no-ops. Status freezes after a restart (a first-class D-8 operation). markLost doesn't call OnSessionEnd. Fix: reconcile re-registers reconnected sessions (grid heuristic at least); decide + document the typed-hook token-across-restart story (the token IS in the 0600 shim-launch.json — re-read it on reconcile to re-share, or an explicit decision).
3. **Real hooks can't drive status — the mapping bridge is missing** (both HIGH): `swarm hook <event>` doesn't parse the stdin JSON payload (reads only key=value args); the claude adapter injects bare `swarm hook Notification` with no args; the engine never consults the session's SignalSources to normalize event→status (session.sources stored, never read). So a real hook posts an empty payload → rejected. The adapters' status tables are dead code. E11.7 scenarios 4/5 pass ONLY in simulation. Fix: the hook parses the JSON payload; the engine normalizes event+payload → status via the registered SignalSources (that is what they are FOR). AND fix the Claude event mapping: PermissionRequest is the dedicated permission event; Notification covers permission/idle/auth/etc. — do not map all Notification → permission.
4. **Adapters dormant** (codex HIGH-required, Opus HIGH-Epic14-deferrable): production detect + skeleton Launch resolve only "fake"; the registry/claude.New/codex.New/SignalSources are imported by zero non-test code. Adjudication: WIRE the adapters into launch NOW (detect via registry; Launch resolves agent→registry→adapter.Command for real argv + hook injection + RegisterSession(SignalSources())) — it is the natural completion and required to exercise the mapping bridge (#3). Test with a fake-binary-backed/stub-prober approach (no billable runs). Only the REAL billable claude/codex RUN stays behind Epic 14's flagged real-CLI smoke.
5. **Resume is a recorded label, not a flow** (both): no validation (endpoint/exists/ended/agent-match/adapter-supports); adapter.Resume never called at runtime (ConversationID never consumed); no TUI resume affordance. Fix: validate the source; call adapter.Resume to compose the resume argv from the source's ConversationID; add a TUI resume affordance (offer on ended/lost rows). R-2/scenario 12.
6. **Codex fixtures use the wrong wire format** (codex): codex.json uses `{type:"exec_approval_request", conversation_id}` but the real app-server is JSON-RPC (`turn/started`, `turn/completed`, `item/commandExecution/requestApproval`, `threadId`/`turnId`). Fix the codex fixtures + event descriptors to the real format (the mapping stays drift-resilient, but fixtures should match reality).
7. **Grid-tap serial HOL blocking** (both MEDIUM): serial sampling stalls up to helloIO=3s per busy shim, blowing L1's cadence for later sessions (safety is fine — no S2/leak/starvation). Fix: short per-tap timeout (< the 200ms poll), sample concurrently, or skip client-attached sessions.
+ GG-5 evidence file at close; RegisterSession passes nil instead of SignalSources() (folded into #3/#4).
