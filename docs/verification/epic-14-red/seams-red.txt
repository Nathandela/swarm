SEAM 1 — a7d: shim arm->spawn ordering (internal/shim/shim_signal_order_test.go)
------------------------------------------------------------------------------
Neutralization: in internal/shim/shim.go's Run(), the block that arms the
self-containment signal handler (signal.Notify) was moved from BEFORE
pty.StartWithSize to AFTER it, and the testHookAfterSignalArm() seam call was
kept BEFORE the (now-later) arming — reproducing the exact "reordered" bug the
test's own comments describe: a signal delivered in the window fires before the
handler is armed, so it is never buffered. A deliberate 100ms sleep was added
after the hook fires (before arming) to make the miss deterministic rather than a
microsecond scheduling race against the next few instructions.

FAIL
FAIL	github.com/Nathandela/swarm/internal/shim	11.337s

EXIT=1
(The window SIGTERM is lost — never buffered — because the handler is armed only
after it fires; the idle test agent has no SIGTERM handler of its own and parks
forever reading stdin; Run never returns, and waitRun's bounded 10s timeout is
what surfaces the failure, exactly as the test's own comments predict for the
reordered-bug case. The test's own SIGTERM backstop keeps this from crashing the
test binary itself.)
